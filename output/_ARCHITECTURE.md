## Namespace Convention

All BalloonFlow code uses a single flat namespace:

```csharp
namespace BalloonFlow
{
    // All runtime classes
}
```

Editor scripts use:
```csharp
namespace BalloonFlow.Editor
{
    // Editor-only classes, wrapped in #if UNITY_EDITOR
}
```

## Singleton Pattern

All core managers inherit `Singleton<T>`. This provides:
- Automatic `DontDestroyOnLoad`
- Duplicate instance detection and destruction
- Null-safe `Instance` property (returns null if application is quitting)
- `HasInstance` static check

### Usage
```csharp
public class MyManager : Singleton<MyManager>
{
    protected override void OnSingletonAwake()
    {
        // Initialization here (NOT in Awake)
    }
}

// Access
MyManager.Instance.DoSomething();
if (MyManager.HasInstance) { ... }
```

| InputHandler | Touch/mouse input processing |
| UIManager | Page visibility management |
| PopupManager | Priority popup queue |

| ScoreManager | Level score and star tracking |

## EventBus Pattern

`EventBus` is a static pub/sub system using struct-based events. All inter-system communication goes through EventBus instead of direct references.

### Usage
```csharp
// Subscribe (in OnEnable or OnSingletonAwake)
EventBus.Subscribe<OnBalloonPopped>(HandleBalloonPopped);

// Unsubscribe (in OnDisable or OnDestroy)
EventBus.Unsubscribe<OnBalloonPopped>(HandleBalloonPopped);

// Publish
EventBus.Publish(new OnBalloonPopped
{
    balloonId = 42,
    color = 3,
    position = transform.position
});

// Handler
private void HandleBalloonPopped(OnBalloonPopped evt)
{
    Debug.Log($"Balloon {evt.balloonId} popped at {evt.position}");
}
```

### Rules
- Events are `struct` types defined in `GameEvents.cs`
- Always unsubscribe in `OnDestroy` to prevent leaks
- Use `EventBus.ClearAll()` on scene transitions if needed
- Never use `Find()` or `FindObjectOfType()` — use EventBus instead

### Event Naming Convention
All event structs follow: `On{Subject}{Verb}` pattern.
- `OnBalloonPopped`, `OnHolderTapped`, `OnScoreChanged`
- `OnBoardCleared`, `OnLevelFailed`, `OnComboIncremented`

## ObjectPool Pattern

`ObjectPool` handles reusable GameObjects. `ObjectPoolManager` is the central registry.

### Usage
```csharp
// Register a pool (usually in level initialization)
ObjectPoolManager.Instance.CreatePool("Balloon", balloonPrefab, 30);
ObjectPoolManager.Instance.CreatePool("Dart", dartPrefab, 20);

// Get from pool
GameObject balloon = ObjectPoolManager.Instance.Get("Balloon");
balloon.transform.position = spawnPos;

// Return to pool
ObjectPoolManager.Instance.Return("Balloon", balloon);

// Return all at level end
ObjectPoolManager.Instance.ReturnAll("Balloon");
```

### Rules
- No `Instantiate()`/`Destroy()` loops at runtime — use pools
- No `new GameObject()` at runtime — use `ObjectPoolManager.Get()` or `Resources.Load()`
- Pools auto-expand by default; set `autoExpand: false` for fixed-size pools
- Pre-configure pools in Inspector via `ObjectPoolManager._poolDefinitions`

## Forbidden Patterns (Runtime Code)

| Pattern | Alternative |
|---------|-------------|
| `new GameObject()` | `ObjectPoolManager.Get()` or `Resources.Load()` |
| `Instantiate()`/`Destroy()` loops | `ObjectPool.Get()`/`Return()` |
| `Find()`, `FindObjectOfType()` | `[SerializeField]` or `EventBus` |
| `Update()` for event logic | `EventBus.Subscribe<T>()` |
| Magic numbers | `const` fields or `[SerializeField]` |
| String comparison for state | Use `enum` types |
| God class (>1000 lines) | Split into partial classes |
| Deep nesting (>3 levels) | Extract methods |
| SDK `using` outside `#if` | Wrap in `#if SDK_SYMBOL` |

## Coding Standards for Sub Coders

1. **Always inherit Singleton<T>** for system-level managers
2. **Always subscribe/unsubscribe** EventBus in lifecycle methods
3. **Always use ObjectPoolManager** for runtime object creation
4. **Always use [SerializeField]** for Inspector-bound references
5. **Always add XML doc comments** on public methods/properties
6. **Always use regions** (#region Fields, Properties, Public Methods, Private Methods)
7. **Never exceed 1000 lines** per file
8. **Never use Update()** for event-driven logic
9. **Never hardcode** colors, sizes, or positions
10. **Never reference** other systems directly — use EventBus

## Star Score Formula

```
base_score      = balloon_count * 100
star_1_threshold = base_score
star_2_threshold = ceil(base_score * 1.5)
star_3_threshold = ceil(base_score * 2.2)
```

Managed by `ScoreManager`. Initialize per level via `ScoreManager.Instance.InitializeLevel(balloonCount)`.
