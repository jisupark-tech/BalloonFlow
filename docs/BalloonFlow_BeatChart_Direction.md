# Beat Chart Direction — Balloon Flow
# Workflow 2단계 디렉션 입력 파일
# 디렉터가 채운 패키지 단위 레벨 디자인 디렉션
# AI는 이 디렉션에 맞춰 Level Library를 생성한다

## 전체 구조
- 프로젝트: Balloon Flow
- 코어 메카닉: queue_sort (큐 또는 홀더에서 보관함 터치 → 보관함이 레일에 올라가 한 바퀴 돌며 매칭 풍선에 다트 발사 → 레일 도중 탄창이 0이 되면 즉시 제거 → 한 바퀴 후 탄창이 남아있으면 홀더로 복귀 대기)
- 총 레벨: 300 (15패키지)
- 패키지 단위: 20레벨
- 실패 조건: 홀더 초과 (Pixel Flow 동일. 홀더에 복귀 보관함이 5개를 초과하면 실패)
- 큐: 보관함 초기 배치 공간. 유저가 터치하면 레일에 올라감
- 홀더: 탄창이 남아 돌아온 보관함 대기 공간. 5칸. 터치하면 레일에 다시 올라감. 초과 시 실패
- 레일: 보관함이 한 바퀴 도는 경로. 도중 탄창 0이면 즉시 제거
- 클리어 조건: board_clear (필드 위 모든 풍선 제거)
- 색상 팔레트: 전체 28색. 레벨마다 자유롭게 선택 (순서/해금 제한 없음)
- 색상 수(num_colors): 한 레벨당 2~11종. 패키지별 점진 증가
- 필드 카테고리 기믹과 큐 카테고리 기믹을 구분. 기믹 명세 참조

### 필수 규칙 (Hard Rules — 위반 절대 불가)

| 항목 | 최소 | 최대 | 비고 |
|------|------|------|------|
| 큐 가로 길이 | 2칸 | 5칸 | |
| 보관함 탄창 (다트 수) | 5 | 50 | 5단위로 설정 (5/10/15/.../50) |

- **색상 수 (num_colors)**: 한 레벨에서 동시에 사용하는 색상 개수. 2~11종. Hard Rule
- **색상 팔레트**: 전체 28색. 1번부터 순차적으로 도입 (팔레트 1번이 가장 먼저 등장, 28번이 가장 나중)
- 순차 도입이지만 "N번 색은 반드시 PKG X에서 해금" 같은 하드 룰은 없음. 자연스러운 흐름으로 진행

### 큐 가로 길이 설계 규칙

**필수 규칙:**
- 기믹 도입(튜토리얼) 레벨에서는 큐 가로 길이를 **2칸으로 줄인다**
  - PF 실측: Line Size 2칸 11회 중 10회가 기믹 도입 레벨 (Lv.1/8/14/70/85/100/120/171/230/290)
  - 새 기믹 학습 시 큐를 좁혀 복잡도를 낮추는 검증된 패턴
- 큐 가로 길이 2~5칸 (Hard Rule)

**권장 규칙:**
- PKG 진행에 따라 평균 큐 가로 길이를 점진적으로 넓힘 (3.0 → 4.2)
- 하드/슈퍼하드 레벨은 노말보다 큐를 넓게 (자원 관리 복잡도 증가)
- 휴식 레벨은 큐를 좁게 (긴장 해소)
- 동일한 큐 가로 길이를 4레벨 이상 연속 사용 금지 (기존 Soft Rule)
- **PKG 테이블의 하한값은 노말 기준.** 튜토리얼(2~3색)과 휴식(3~4색)은 어떤 PKG에서든 색상을 낮출 수 있음. PKG 15의 튜토리얼도 2색 가능

- **희소 기믹 (Wall, Frozen Dart)**: 도입 후 낮은 빈도. 등장만으로 긴장감
- **마지막 기믹 (Color Curtain)**: 도입 PKG에서만 사용. 학습 구간만 존재

| 필드 크기 | — | 제한 없음 (피어 분석 후 확정) | PF 후반 약 27×32. 프로토타입에서 실측 필요 |
| 레벨당 기믹 종류 수 | 0 | 5종 | 레벨 내 동시 사용 기믹 최대 5종 |

- Chain(보관함 연결) 기믹은 연결 수(2/3/4/5)와 무관하게 **1종으로 취급**. Dual~Penta 중 하나라도 사용되면 "Chain 사용", 모두 미사용이면 "Chain 미사용"

### 권장 규칙 (Soft Rules — 예외 허용하되 최소화)

**기믹 연속 등장 제한:**
- 동일 기믹을 **6레벨 이상 연속** 등장시키지 않는다
- 예: Lv.105~109에 Hidden이 5레벨 연속 등장하면 Lv.110에서는 반드시 Hidden을 빼야 함 (또는 109 이전 중간에 빼기)

**기믹 망각 방지:**
- 기믹이 사용된 후, **30레벨 이내에 다시 사용**되어야 한다
- 예: Lv.80에서 Chain이 마지막으로 등장했으면, Lv.110 이전에 Chain이 다시 등장해야 함
- 목적: 유저가 기믹 동작 규칙을 잊지 않도록 주기적으로 환기

**파라미터 연속 반복 제한:**
- 큐 가로 길이, 필드 가로 길이, 색상 수가 **동일한 값으로 4레벨 이상 연속** 사용 불가
- 예: Lv.192~194가 큐 4칸이면, Lv.195는 반드시 큐 4칸이 아닌 값이어야 함 (또는 194 이전 중간에 다른 값 넣기)
- 목적: 레벨 간 변화를 유지하여 단조로움 방지

## 레벨 목적 분류 (purpose_type)
- 튜토리얼: 메카닉/기믹 학습. 가이드 제공. 목표 CR 88~95%. **색상 2~3색** (기믹 학습에 집중)
- 노말: 일반 플레이. 적절한 고민과 쾌감. 목표 CR 40~80% (PKG 진행에 따라 하락)
- 하드: 도전. 순서 전략이 핵심. 실패 후 재도전 유도. 목표 CR 20~60% (후반 PKG에서 20%대)
- 슈퍼하드: 패키지 보스. 강한 성취감. 과금 포인트. 목표 CR 12~45% (후반 PKG에서 12%대)
- 휴식: 하드/슈퍼하드 직후 배치. 긴장 해소. 목표 CR 60~90%. **색상 3~4색** (쉬어가는 구간)
- **난이도 순서 (어려운 순): 슈퍼하드 > 하드 > 노말 > 휴식 > 튜토리얼.** CR은 튜토리얼이 가장 높고 슈퍼하드가 가장 낮음

### purpose_type별 색상 수 가이드
- 튜토리얼: **2~3색**. 신규 기믹에 집중시키기 위해 색상 최소화. PF도 기믹 도입 레벨에서 색상 2~4로 낮춤 (100레벨 이후에도)
- 휴식: **3~4색**. 긴장 해소. 색상 부담 최소
- 노말: PKG에 따라 점진 상승. 신규 기믹 도입 직후 노말도 3~4색으로 낮출 수 있음
- 하드/슈퍼하드: 해당 PKG의 색상 상한 근처. 기믹 + 색상 동시 압박
- 핵심: **신규 요소(기믹/색상) 도입 시 다른 축(색상/기믹)을 줄여서 총 난이도를 조절**

- 하드: UI 색상 변경 (경고감을 주는 톤). 유저에게 "이 레벨은 어렵다"는 시각적 신호
- 슈퍼하드: UI 색상 추가 변경 (긴장감을 주는 톤). 하드보다 더 강한 시각적 신호
- 튜토리얼/휴식: 노말과 동일
- 목적: 레벨 시작 전에 난이도를 즉시 인지하여 부스터 선제 사용 판단을 유도. 과금 포인트와 연결

## 패키지 내 포지션 규칙 (모든 패키지 공통)
### 하드 레벨 배치
- 패키지 내 pos 4, 9, 14, 19에 하드 또는 슈퍼하드 배치 (PKG 2부터 적용)
- PKG 1은 학습 구간이라 pos 19에만 하드 1개 (pos 4, 9, 14는 노말)
- 슈퍼하드: PKG 2부터 pos 19가 슈퍼하드. PKG 1은 pos 19도 일반 하드

### 휴식 레벨 배치
- 모든 하드/슈퍼하드 직후에 반드시 휴식 레벨 배치
- PKG 2부터: pos 5, 10, 15, 20이 휴식 (하드 pos 4,9,14,19의 직후)
- PKG 1: pos 20만 휴식 (pos 19 하드 직후)

### 기믹 도입 레벨
- 패키지 내 pos 1 또는 pos 11에 배치
- 기믹 도입 레벨은 튜토리얼로 분류
- 한 패키지에 최대 2개 기믹 도입 가능 (pos 1 + pos 11)

### 노말 레벨
- 나머지 포지션은 노말
- 노말 3개 → 하드 → 휴식의 톱니 리듬

## 난이도 원칙
- **클리어율(CR)**: 낮을수록 어려움. 슈퍼하드 12~45% < 하드 20~60% < 노말 40~80% (PKG 진행에 따라 전체적으로 하락)
- **시도 횟수(Attempts)**: 높을수록 어려움. 슈퍼하드 5~8회 > 하드 3~5회 > 노말 1~2회
- **난이도 순서 (패키지 내 반드시 준수, 어려운 순)**: 슈퍼하드 > 하드 > 노말 > 휴식 > 튜토리얼. 모든 지표(CR, 시도횟수, 기믹 수)에서 이 순서 유지. 노말이 하드보다 어렵거나, 하드가 슈퍼하드보다 어려운 경우 절대 불가
- 톱니(sawtooth) 패턴: 노말로 서서히 올라감 → 하드에서 정점 → 휴식에서 급격히 해소 → 반복
- 신규 색상/기믹 도입 시 오브젝트 수를 직전 대비 줄여서 복잡도 증가와 물량 증가를 동시에 주지 않음

## 기믹 도입 원칙
- 기믹은 패키지 내 pos 1 또는 pos 11에서 도입
- 기믹 도입 레벨은 튜토리얼 레벨로 분류 (가이드 제공, 높은 CR)
- **기믹 격리 원칙**: 도입 레벨에서는 해당 신규 기믹만 등장, 기존 기믹 미등장. 유저가 새 기믹 규칙을 순수하게 학습
- 도입 후 2~3레벨은 해당 기믹만 사용하여 적응 구간 제공, 이후 기존 기믹과 혼합

## 기믹 빈도 가이드라인

### 레벨 유형별 기믹 종수 제한
- 슈퍼하드: 3~5종 허용 (전체 기믹 혼합 가능)
- 하드: 2~3종 허용
- 노말: 2종 이하 권장
- 튜토리얼: 1종 (도입 기믹만)
- 휴식: 0~1종 (있더라도 가장 가벼운 것만)

### 레벨 내 비율 (피어분석 후 확정)
- 각 기믹이 레벨 내에서 차지하는 비율(오브젝트/보관함 대비)은 피어분석 후 확정
- PF Deconstruction 데이터에서 기믹별 사용 패턴 추출 후 가이드라인 설정 예정
- 현재 제한 없음. 기믹 명세의 난이도 레버 참조

### 패키지 내 등장 빈도 (피어분석 후 확정)
- 20레벨 중 해당 기믹이 등장하는 레벨 수는 피어분석 후 확정
- PF 기믹 수 가이드라인(위 테이블) 참고하여 전체 밀도는 조절

### 빈도 원칙
- 기믹 도입 패키지에서는 낮은 빈도. 유저가 적응할 시간을 줌
- 후반 패키지로 갈수록 등장 빈도 증가
- 기믹 6레벨 이상 연속 등장 금지 (Soft Rule)
- 기믹 사용 후 30레벨 이내 재등장 (Soft Rule, 망각 방지)

### 패키지 의도
전체 기믹 혼합 극한 → 최종보스(pos 19 슈퍼하드) → pos 20 해소. 모든 요소 조합 최고 난이도

### 난이도 구성
- 튜토리얼: 0개
- 노말: 12개 (pos 1-3, 6-8, 11-13, 16-18)
- 하드: 3개 (pos 4, 9, 14)
- 슈퍼하드: 1개 (pos 19)
- 휴식: 4개 (pos 5, 10, 15, 20)

### 목표 지표
- CR: 하한 30% / 평균 58% / 상한 92%
- 시도횟수: 하한 1.0 / 평균 3.2 / 상한 7.0

### 색상 수
- 하한: 6 / 평균: 7.6 / 상한: 8
- 색상 분포: 6색:8색 = 15%:85%
- 색상 비율 의도: 거의 균등. 각 색상 10~15%. 보스 런업(pos 16-18)에서 의도적 편중으로 "함정"

### 오브젝트 수
- 피어분석 후 확정. PF PKG5 참고. PF 후반 필드는 27×32 규모

### 기믹 수 (레벨당 기믹 종류 수)
- 하한: 0 / 평균: 2.3 / 상한: 5
- 등장 기믹: 5종 전체
- 기믹 등장 레벨: 약 15~17개 / 20레벨 (휴식+일부 노말만 기믹 없음)

### 신규 요소 도입
- 없음 (기존 5종 조합으로 난이도 구성)

### 과금 포인트
- 없음 (도전과 성취 중심)

## AI 지시사항
### 레벨 설계 규칙 (필수 준수)
- 필수 규칙(Hard Rules): 큐 가로 2~5, 색상 2~11, 필드 50×50 이하, 레벨당 기믹 5종 이하. Chain은 연결 수 무관 1종 취급
- 권장 규칙(Soft Rules): 기믹 6레벨 이상 연속 등장 금지, 30레벨 이내 재등장, 파라미터 4레벨 이상 연속 동일값 금지

### 패키지 내 포지션 규칙 (필수 준수)
- PKG 2~5: pos 4, 9, 14, 19에 하드 또는 슈퍼하드. pos 5, 10, 15, 20에 휴식
- PKG 1 예외: pos 19에만 하드. pos 20에 휴식. 나머지는 튜토리얼+노말 (학습 구간)
- pos 1 또는 11: 기믹 도입 시 튜토리얼 (해당 패키지에 신규 기믹이 있을 때만)
- 나머지: 노말

### 난이도 순서 (필수 준수, 어려운 순)
- 패키지 내에서 슈퍼하드 > 하드 > 노말 > 휴식 > 튜토리얼 (어려운 순). CR은 낮을수록 어려움, 시도 횟수는 높을수록 어려움. 튜토리얼은 색상 2~3색, 휴식은 3~4색
- 모든 지표(CR, 시도횟수, 오브젝트 수, 기믹 수)에서 이 순서 유지
- 위반 절대 불가

### 기믹 빈도 (필수 준수)
- 레벨 유형별 기믹 종수 제한: 슈퍼하드 3~5종, 하드 2~3종, 노말 2종 이하, 튜토리얼 1종, 휴식 0~1종
- 레벨 내 비율과 패키지 내 등장 빈도는 "기믹 빈도 가이드라인" 테이블 범위 안에서
- 상한 초과는 전체 레벨의 5% 이내 예외만 허용

### 기믹 격리 (필수 준수)
- 기믹 도입 레벨: 해당 신규 기믹만 등장, 기존 기믹 미등장
- 도입 후 2~3레벨: 해당 기믹만 사용 (적응 구간)
- 이후: 기존 기믹과 혼합 허용

### 생성 규칙
- 각 패키지 디렉션의 범위(CR, 시도횟수, 색상 수, 오브젝트 수, 기믹 수) 안에서 Level Library를 채울 것
- 색상 분포와 색상 비율 의도를 반드시 참고하여 레벨별 색상 수 결정
- 신규 색상/기믹 도입 레벨에서는 오브젝트 수를 직전 대비 줄일 것
- 실패 조건: 홀더 초과. holder_slots = 5 고정
- 총 레벨: 300. 총 패키지: 15. PKG 6~15 상세는 기믹 확정 후
- 색상 수: 레벨당 2~11종. 색상 팔레트 28색에서 자유 선택 (해금 개념 없음)
- 큐 = 초기 배치, 홀더 = 복귀 대기. 구분하여 사용
- emotion_curve와 designer_note는 레벨별로 반드시 채울 것
