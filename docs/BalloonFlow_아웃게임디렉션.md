# 1차 프로젝트(리스킨) 기준 최소 수익 루프

## 전제
- 실패 조건: 홀더 초과 (Pixel Flow 동일). 실패 조건이 레일 초과로 변경 시 부스터 구성도 변경됨
- 1.0(소프트런칭) 범위만 정의. 이후 업데이트(피기뱅크, 시즌패스 등)는 소프트런칭 데이터 확인 후
- 부스터 4종 확정 (홀더 초과 기준). 실패 조건 변경 시 부스터 재설계 필요

## 1. 라이프 시스템

### 기본 구조
- 유형: 하트
- 최대 수: 5개
- 소모: 레벨 실패 시 1개 (클리어/취소 시 소모 없음)
- 자동 충전: 30분에 1개
- 만충전 시간: 2시간 30분

### 충전 경로
- 시간 경과 (무료)
- 보상형 광고 시청 (1개 충전. 하트 4개 이하일 때 노출 가능)
- 코인으로 즉시 전체 충전 (900코인)
- IAP로 충전 (상점)

### 디자인 의도
- 하트 0이 되는 시점이 과금/광고 접점. "지금 그만둘지, 하트를 채울지" 선택
- 30분 충전은 "잠깐 쉬었다 오면 1개 있다"는 자연스러운 복귀 유도
- Pixel Flow, Royal Match, Candy Crush 등 업계 표준과 동일

## 2. 부스터

### 구조
- 총 4종
- 카테고리: in_play(판 진행 중) 4종 전부
- continue(이어하기)는 별도 섹션에서 정의
- 해금 시점: PF 참고 순차 해금. 트레이 추가(Lv.10), 손(Lv.12), 셔플(Lv.15), 색상 제거(Lv.18)
- 사용 제한: 현재 제한 없음 (피어 분석 후 제한값 입력 예정)

### 부스터 4종 (홀더 초과 기준)

**부스터1: 트레이 추가 (in_play) — 해금 Lv.10**
- 효과: 레일의 트레이를 1개 추가. 레일에서 동시에 작동할 수 있는 보관함 수가 늘어남
- 사용 시점: 판 진행 중 원하는 타이밍에 사용
- 가격: 300코인 / 1개
- 디자인 의도: 레일 운용 여유를 늘려 홀더 초과 위험을 간접 완화. 가장 저렴한 "가벼운 보험"
- 참고: 홀더는 5칸 고정. 이 부스터는 홀더가 아니라 레일 트레이를 확장하는 것
- PF 대응: Extra Tray Slot (PF Lv.7 → BF Lv.10)

**부스터2: 손 (in_play) — 해금 Lv.12**
- 효과: 큐에서 원하는 보관함을 선택하여 레일에 올릴 수 있음. 큐의 순서를 무시하고 지정 선택
- 사용 시점: 판 진행 중 원하는 타이밍에 사용
- 가격: 1,900코인 / 3개
- 디자인 의도: 큐 순서 의존을 깨고 전략적 개입 가능. "지금 이 색이 필요한데 큐 앞쪽에 없다"는 상황에서 직접 선택
- PF 대응: Select Tool (Lv.12)

**부스터3: 셔플 (in_play) — 해금 Lv.15**
- 효과: 큐에 대기 중인 보관함의 배치 순서를 무작위로 재배치
- 사용 시점: 판 진행 중 원하는 타이밍에 사용
- 가격: 1,500코인 / 3개
- 디자인 의도: 큐 순서가 막혔을 때 리커버리. 운 요소 포함
- PF 대응: Shuffle (Lv.15)

**부스터4: 색상 제거 (in_play) — 해금 Lv.18**
- 효과: 유저가 색상 하나를 지정하면, 해당 색상의 풍선(필드) + 보관함(큐/홀더) + 다트(레일) 모두 일괄 제거
- 사용 시점: 판 진행 중 원하는 타이밍에 사용
- 가격: 2,900코인 / 3개
- 디자인 의도: 가장 강력하고 비싼 부스터. 필드와 큐/홀더를 동시에 정리. 실질적으로 색상 수를 1개 줄이는 효과
- PF 대응: Super Shooter (Lv.18)

### 가치 위계
- 트레이 추가(300) < 셔플(1,500) < 손(1,900) < 색상 제거(2,900)
- 가격이 효과의 강도를 반영. 트레이 추가는 가볍고 저렴한 보험, 색상 제거는 판을 뒤집는 결정적 아이템

### 실패 조건 변경 시 부스터 변경 예상
- 레일 초과 방식이면 부스터1(트레이 추가)의 역할이 달라짐. 레일 구조 자체가 바뀌므로 재설계 필요
- 부스터2(손)는 Hidden 기믹 대응이라 실패 조건과 무관하게 유지
- 부스터3(셔플), 부스터4(색상 제거)는 실패 조건과 무관하게 유지 가능

### 획득 경로
- 레벨 클리어 보상 (저확률)
- 코인으로 구매 (상점)
- 보상형 광고 시청 (1회 무료 사용)
- IAP 번들에 포함

### 가격 정책
- 코인 가격은 부스터 가격(300~2,900코인)과 이어하기 1회차(900코인)를 기준으로 역산
- 표준 코인 상품 Tier 1의 코인 지급량이 이어하기 N회로 환산 가능하도록 설계
- 부스터는 "더 쉽게 클리어하는 파워". 시즌패스의 꾸미기 보상과 성격이 다름
- 무과금 유저가 자연 획득 코인으로 트레이 추가(300)는 자주, 색상 제거(2,900)는 가끔 살 수 있는 밸런스

## 3. 컨티뉴 (이어하기)

### 기본 구조
- 실패 시 "이어하기" 팝업 노출
- 첫 실패 시: 무료 이어하기 1회 제공
- 이후 비용 (동일 플레이 회차 내 단계별 상승):
  - 1회차: 900코인
  - 2회차: 1,900코인
  - 3회차: 2,900코인 (최대)
- 해당 플레이를 포기하고 재시도하면 1회차(900코인)부터 다시 시작
- 이어하기 시: 홀더의 보관함을 이어하기 대기열로 이동. 유저가 터치할 때까지 이어하기 대기열에서 머무르고, 터치하면 레일에 올라감. 자동으로 홀더에 내려오지 않음
- 사용 제한: 현재 제한 없음 (피어 분석 후 제한값 입력 예정)
- 동영상 광고 이어하기 지면: 추후 추가 (피어 분석 필요)

### 디자인 의도
- "아깝게 실패"한 유저의 재도전 욕구를 코인 소비로 연결
- 단계별 가격 상승(900→1,900→2,900)으로 반복 이어하기의 비용이 급격히 증가. 3회 연속 이어하기(총 5,700코인)는 매우 비싸서 재시도를 유도
- 재시도하면 900코인으로 리셋되므로, "포기하고 다시 도전"이라는 건강한 루프가 유지됨
- near_miss_rate가 높은 레벨에서 컨티뉴 사용률이 높아야 좋은 설계
- 하드/슈퍼하드에서 컨티뉴 수요가 집중되는 것이 정상

## 4. 재화

### 구조
- 코인 1종 (소프트 재화)
- 하드 재화(젬)는 1.0에 미포함. 피어 분석 후 필요 시 추가
- 초기 하트: 5개 (최대치에서 시작)
- 신규 유저 초기 코인: 1,000코인 (이어하기 약 1회분. 첫 출시에는 라이브옵스 이벤트가 부족하여 코인 수급이 어려우므로 여유있게 제공)

### 소스 (획득)
- 레벨 클리어 보상 (기본 소스): 노말 50코인, 하드 75코인, 슈퍼하드 100코인
- 보상형 광고 시청
- IAP 구매 (코인팩)
- 데일리 보상

### 싱크 (소비)
- 부스터 구매
- 하트 전체 충전 (900코인)
- 컨티뉴 (900 → 1,900 → 2,900코인. 동일 회차 내 단계별 상승)
- (이후) 스킨/꾸미기

### 밸런스 원칙
- 무과금 유저도 자연스러운 플레이로 부스터를 가끔 살 수 있는 수준의 코인 지급
- 하드/슈퍼하드에서 부스터+컨티뉴를 쓰면 자연 획득보다 소비가 많아지는 구조 (과금 유도)
- net_coin_flow 추적: 패키지 단위로 소스 > 싱크(인플레) 안 되도록 관리

## 5. 상점 (IAP)

### 공통 원칙
- 1.0(소프트런칭)은 Static Pricing(고정 가격) 상품만. Dynamic Pricing은 소프트런칭 데이터 확인 후 도입
- 모든 상품의 가치 기준점: 표준 코인 상품 Tier 1의 효율 = 1.0배

### 단일 가치 시스템
- 모든 IAP 상품의 가치는 총가치(유저 관점 인지 가치)라는 단일 기준으로 관리
- 총가치 = 모든 아이템을 할인 없이 개별 판매가 기준으로 계산한 총합
- 상품 설계의 기준이 되는 모든 계산(총가치, 할인율, 효율)은 총가치 기반

### 가치 계산 공식
- 총가치 효율 = 상품의 총가치($) / 판매 가격($)
- 총가치 할인율 = 1 - (판매 가격($) / 상품의 총가치($))
- 코인 기준 효율 = 제공 코인 총가치($) / 판매 가격($)
- 코인 기준 할인율 = 1 - (판매 가격($) / 제공 코인 총가치($))
- 코인 비율 = 제공 코인 총가치 / 상품의 총가치
- 이어하기 횟수 환산 = 제공 코인 / 900 (이어하기 1회차 = 900코인 기준)

### 직관적 가치 설계 원칙
- 핵심 연동: 가장 저렴한 코인 상품(Tier 1)은 이어하기 1회차(900코인)를 즉시 구매할 수 있는 양 제공
- 코인을 포함하는 모든 상품의 코인량은 이어하기 1회차 0.5회(450코인)의 배수
- 100, 500, 1000 등 딱 떨어지는 명료한 숫자 단위 우선
- 위 규칙을 만족시킬 수 없는 가격대에서는 해당 상품 라인업을 구성하지 않음

### 상품 테이블 필수 컬럼 (AI 생성 시 준수)
| 가격($) | 상품명 | 코인 기준 할인율(%) | 총가치($) | 총가치 할인율(%) | 총가치 효율 | 코인 비율 | 코인 기준 효율 | 코인 | 부스터 구성 | 이어하기 횟수 환산 |

### 아이템 컬럼 작성 규칙
- 부스터 4종 전체 포함 시: "부스터 4종 각 N개"
- 일부만 포함 시: 아이템명 명시 (예: "트레이 추가 2개, 셔플 3개")
- "부스터 3종 각 1개"처럼 어떤 아이템인지 불명확한 표현 금지

### 5-1. 표준 코인 상품

| 구분 | 설정값 | 비고 |
|------|--------|------|
| 가격 범위 | $0.99 ~ $99.99 | 글로벌 표준 가격대 |
| 티어 수 | 6단계 | $0.99, $2.99, $4.99, $9.99, $19.99, $99.99 |
| 효율 | 1.0배 ~ 2.0배 | Tier 1 = 1.0배 기준, 상위 티어 점진 상승 |
| 보상 구성 | 코인 100% | 다른 아이템 미포함. 기준점 역할 유지 |
| dynamic_pricing | False | |
| 해금 조건 | Lv.1+ | |
| 구매 제한 | 없음 | 상시 구매 |
| 노출 | 상점 상시. 저가→고가 순 정렬 | |

- 설계 원칙: Tier 1($0.99) = 이어하기 1회차(900코인). 경제의 기준점
- 상위 티어: "더 많이 결제할수록 더 높은 효율" 원칙 준수
- 코인량은 100, 500, 1000 등 명료한 숫자 단위 우선

### 5-2. 표준 번들 상품

| 구분 | 설정값 | 비고 |
|------|--------|------|
| 가격 범위 | $4.99 ~ $99.99 | |
| 티어 수 | 5단계 | $4.99, $9.99, $19.99, $49.99, $99.99 |
| 종합 효율 | 1.7배 ~ 2.5배 | 동일 가격대 코인 상품보다 항상 높아야 함 |
| 코인 기준 효율 | 1.0배 ~ 1.3배 | 핵심 가치는 코인이 아니라 부스터 |
| 보상 구성 | 코인 + 부스터 조합 | |
| dynamic_pricing | False | |
| 해금 조건 | Lv.1+ | |
| 구매 제한 | 없음 | |
| 노출 | 상점 상시 | |

- 설계 원칙: 가치 누적 구조(cumulative). 상위 티어는 하위 티어 구성 포함 + 수량 증가 또는 상위 아이템 추가
- 번들에 포함된 코인의 기준 효율은 1.0~1.3배로 낮게 설정하여, 핵심 가치가 코인이 아닌 부스터임을 명확히 함
- 부스터 구성 가이드:
  - $4.99: 부스터 4종 각 1개 + 코인
  - $9.99~$19.99: 부스터 4종 각 2~3개 + 코인
  - $49.99~$99.99: 부스터 4종 대량 + 코인
- 아이템 컬럼 규칙: 4종 전부 포함 시 "부스터 4종 각 N개". 일부만 포함 시 아이템명 명시 (예: "트레이 추가 2개, 셔플 3개")

### 5-3. 스타터팩 (첫 구매 유도)

| 구분 | 설정값 | 비고 |
|------|--------|------|
| 가격 | $0.99 ~ $1.99 | 1~2단계 |
| 종합 효율 | 3.0배+ | 1회 한정이므로 최고 효율 허용 |
| 보상 구성 | 코인 + 부스터 소량 | |
| dynamic_pricing | False | |
| 해금 조건 | Lv.10~20 부근 (피어 분석 후 확정) | |
| 구매 제한 | 1회 한정 | |
| 노출 | 해금 시 팝업 + 상점 한정 배너 | |

- 설계 원칙: 첫 결제 장벽을 낮추는 것이 목적. "이 가격에 이 구성이면 안 살 이유가 없다"

### 5-4. 광고 제거

| 구분 | 설정값 | 비고 |
|------|--------|------|
| 가격 | $2.99 ~ $4.99 (피어 분석 후 확정) | |
| dynamic_pricing | False | |
| 해금 조건 | Lv.20+ | 전면 광고 시작 시점과 동일 |
| 구매 제한 | 1회 (영구) | |
| 노출 | 상점 상시 + 전면 광고 노출 시 "광고 제거" 버튼 | |

### 상품 간 가치 위계 원칙
- 효율: 스타터팩(3.0배+) > 번들(1.7~2.5배) > 코인(1.0~2.0배)
- 스타터팩은 1회 한정이라 최고 효율 허용
- 번들은 코인 상품보다 항상 높아야 함
- 코인 상품이 모든 가치 평가의 기준점(1.0배)
- 상위 가격 상품은 하위 가격 상품보다 총가치가 반드시 높아야 함 (효율이 역진되더라도)

### 이후 업데이트에서 추가될 상품 (1.0 미포함)
- Best Value (Dynamic Pricing, $1~$5 5단계, 상시 최고 효율 2.5배)
- Mega Sale (Dynamic Pricing, NPU 첫 구매 전환 팝업)
- 피기뱅크 (Static, $2.99, 플레이하면 코인 누적 → 구매하여 열기)
- 프리미엄 패스 (Static, 시즌패스)
- 일일 행사 (Static)
- 주간 행사, 주말 세일, 어려운 레벨 번들 (Dynamic Pricing)

## 6. 광고

### 보상형 광고 (Rewarded)
- 하트 충전 (하트 4개 이하일 때)
- 부스터 1회 무료 사용 (판 진행 중)
- 컨티뉴 광고 지면: 추후 추가 (피어 분석 필요)
- 코인 보상 (데일리 보상 2배)

### 전면 광고 (Interstitial)
- 노출 조건: Lv.20까지는 전면 광고 미노출 (FTUE 보호)
- 노출 타이밍: 레벨 결과 화면에서만 노출 (플레이 중, 로비, 상점 등에서는 절대 미노출)
- 노출 후 딜레이: 전면 광고 1회 노출 후 최소 30초간 다음 전면 광고 미노출
- 광고 제거 IAP 구매 시 전면 광고 영구 제거

## 8. 상품 노출 정책

### 팝업 노출 기본 원칙
- 스페셜 오퍼 팝업 노출 후 최소 20분간 다른 스페셜 오퍼 팝업 미노출 (노출 피로도 관리)
- 예외: 각 상품이 해금 조건 달성으로 처음 등장할 때는 쿨타임 무시 (첫 등장 안내)

### 레벨 클리어/실패 후 팝업 노출 우선순위 (1.0 범위)
1. 스타터팩 (NPU에게 1회 한정 노출)
2. 전면 광고 (Lv.20+, 30초 딜레이 준수)

### 상점 내 상품 노출 순서 (1.0 범위)
- 1.0에서는 NPU/PU 분기 미적용 (Dynamic Pricing 미도입)
- 노출 순서:
  1. 스타터팩 (구매 전 한정 배너)
  2. 번들 상품 (저가→고가 순)
  3. 코인 상품 (저가→고가 순)
  4. 광고 제거

### 이후 업데이트에서 추가될 노출 정책
- NPU/PU 분기 노출 (Best Value 도입 시)
- More Offer 버튼 (상품 수 증가 시)
- 시간제 상품 마지막 기회 팝업 (유효 시간 1시간 남았을 때)
- 레벨 실패 후 팝업 우선순위 확장 (Hard Level Bundle 등)

## AI 지시사항
- 위 구조를 기반으로 시스템 설계서의 OutGame 도메인을 채울 것
- "피어 분석 후 확정"으로 표기된 항목은 TBD로 두고 구조만 설계
- 부스터 4종 확정: 트레이 추가(Lv.10, 300코인/1개), 손(Lv.12, 1,900코인/3개), 셔플(Lv.15, 1,500코인/3개), 색상 제거(Lv.18, 2,900코인/3개). PF 참고 순차 해금
- 실패 조건이 홀더 초과임을 전제로 컨티뉴(홀더 비우기), 부스터 설계
- 재화는 코인 1종만. 젬/다이아 미포함
- 1.0 범위(Static Pricing)와 이후 업데이트(Dynamic Pricing)를 명확히 구분
- 상품 테이블 생성 시 필수 컬럼(가격, 상품명, 코인 기준 할인율, 총가치, 총가치 할인율, 총가치 효율, 코인 비율, 코인 기준 효율, 코인, 부스터 구성, 이어하기 횟수 환산) 모두 포함
- 가치 계산 공식에 따라 모든 계산 컬럼 값을 산출하고 교차 검증
- 직관적 가치 설계 원칙 준수: 코인량은 이어하기 0.5회(450코인) 배수, 명료한 숫자 단위 우선
