# Balloon Flow — 기믹 명세 (Level Blockers Overview)

## 공통 용어
- **필드 (Field)**: 상단 풍선 배치 영역. 모든 풍선을 제거하면 클리어
- **큐 (Queue)**: 보관함 초기 배치 공간. 유저가 터치하면 보관함이 레일에 올라감
- **레일 (Rail)**: 보관함이 한 바퀴 도는 경로. 경로 위에서 매칭 풍선을 만나면 다트 발사
- **홀더 (Holder)**: 레일에서 탄창이 남아 돌아온 보관함이 대기하는 별도 공간. 5칸. 터치하면 레일에 다시 올라감. 대기 보관함이 5개를 초과하면 실패
- **이어하기 대기열**: 이어하기 사용 시 홀더의 보관함이 이동하는 임시 공간. 홀더에 종속. 터치하면 레일에 올라감. 자동으로 홀더에 내려오지 않음
- **보관함**: 큐 또는 홀더에서 대기 → 유저 터치 → 레일에 올라가 한 바퀴 돌며 매칭 풍선에 다트 발사 → 레일 도중 탄창 0이 되면 즉시 제거 → 한 바퀴 후 탄창이 남아있으면 홀더로 복귀 대기
  - **탄창 (다트 수)**: 보관함에 발사할 수 있는 다트 숫자가 표시됨. 5단위로 설정. 최소 5 ~ 최대 50
- **다트**: 레일 위 보관함에서 발사되어 풍선을 타격하는 발사체. 1:1 매칭 (같은 색 다트 → 같은 색 풍선)
- **터치 가능 상태**: 유저가 해당 보관함을 탭할 수 있는 상태 (큐 또는 홀더에 있는 보관함)
- **기믹 카테고리**: 필드 / 큐 / 양쪽

## 0. Standard Balloon (기본 풍선)

**도입**: Lv.1 (PKG 1) | **카테고리**: 필드 | **PF 대응**: Standard Pixel (Lv.1)

### 동작 규칙
- 기본 파괴 단위. 색상을 가짐
- 같은 색 다트 1발에 팡 터짐
- 1:1 매칭의 코어 규칙을 형성

### 난이도 레버
- 색상 수 (2~11종)
- 풍선 배치 위치 및 밀도
- 색상별 풍선 수 비율

### 피드백
- 이펙트: 색상 파편 + 반짝 파티클
- 사운드: 팡! (밝고 짧은 팝)
- **P0 필수 피드백**

## 1. Hidden (숨겨진 보관함)

**도입**: Lv.11 (PKG 1) | **카테고리**: 큐 | **PF 대응**: Surprise Shooter (Lv.8)

### 동작 규칙
- 보관함의 색상이 ?로 숨겨져 있음
- 터치 가능 상태가 되면 색상이 공개됨
- 공개 후에는 일반 보관함과 동일하게 동작

### 난이도 레버
- Hidden 보관함 비율 (전체 보관함 중 Hidden 수)
- Hidden 위치 (큐의 앞줄 vs 뒷줄)

### 디자인 의도
- 정보 불확실성 (큐). 전체 순서를 미리 계획하지 못하게 만들어, 불완전한 정보 아래에서 단기 판단을 강제
- 부스터 연결: 손(Select Tool)이 Hidden 상태에서도 선택 가능하여 부스터 가치 상승

### 피드백
- 이펙트: ? → 실제 색으로 전환
- 사운드: 띵! (공개음)
- **P2**

## 2. Chain (연결 보관함)

**도입**: Lv.21 (PKG 2) | **카테고리**: 큐 | **PF 대응**: Connected Shooters (Lv.14)

### 동작 규칙
- 2~4개의 보관함이 연결되어 반드시 함께 발동
- 하나를 탭하면 A→B(→C→D) 순서로 레일에 올라감
- 각 보관함은 독립적으로 레일을 돌며 매칭 풍선에 다트 발사
- 탄창 0인 보관함은 즉시 제거. 탄창이 남은 보관함은 홀더로 복귀
- 연결 수(2/3/4)는 파라미터. Chain은 1종으로 취급

### 난이도 레버
- 연결 수 (2연결 vs 3연결 vs 4연결)
- 연결된 보관함의 색상 조합 (같은 색 vs 다른 색)
- Chain 쌍 수

### 디자인 의도
- 순서 강제 (큐). 연결된 보관함이 한꺼번에 발동하므로, 매칭 풍선 부족 시 여러 칸이 동시에 홀더로 복귀하여 홀더 초과 위험을 만듦
- 잘못된 타이밍에 보내면 즉시 위기. "지금 보낼까 기다릴까"의 긴장감

### 피드백
- 이펙트: 연결된 보관함이 순서대로 슬라이드
- 사운드: 슉슉! (연결 출발음)
- **P2**

## 3. Piñata (피냐타)

**도입**: Lv.31 (PKG 2) | **카테고리**: 필드 | **PF 대응**: Hard Pixels (Lv.30)

### 동작 규칙
- 비풍선 오브젝트. 색상 + 라이프를 가짐 (예: 빨강, 라이프 4)
- 1×1, 2×2 등 크기 가능
- 같은 색 다트로 직접 타격 시 라이프 -1. 매 히트마다 금이 가는 연출
- 라이프 0 → 빠악 터짐. 해당 위치의 필드가 빈 공간이 됨
- 1:1 매칭 유지: 같은 색 다트 → 같은 색 피냐타
- Piñata Box(Lv.161)와의 관계: Piñata가 기본형, Piñata Box가 확장형

### 난이도 레버
- 크기 (1×1, 2×2 등)
- 라이프 (3~10)
- 배치 위치 (경로 차단 여부)
- Piñata 수
- 색상 (희소 색이면 더 어려움)

### 디자인 의도
- 자원 압박 (필드). 해당 색 다트를 여러 발 소비해야 하므로 자원 배분 판단 필요
- 풍선이 아닌 비풍선이므로 다중 히트가 자연스러움. "풍선인데 왜 안 터져?" 위화감 없음
- 풍선(팡!) vs 피냐타(쾅쾅→빠악) 피드백 차별화로 게임 리듬에 변화
- Lv.31에서 첫 피냐타 학습 → Lv.161에서 Piñata Box로 확장

### 피드백
- 이펙트 (타격): 금 가는 균열 + 흔들림
- 사운드 (타격): 쾅! (둔탁한 타격음)
- 이펙트 (파괴): 빠악 터지며 색종이 폭발
- 사운드 (파괴): 빠악! + 축하 사운드
- **P1** (파괴 이펙트), **P2** (타격 이펙트)

## 4. Spawner (보관함 방출기)

**도입**: Lv.41 투명 (PKG 3) / Lv.141 불투명 (PKG 8) | **카테고리**: 큐 | **PF 대응**: Shooter Pipe (Lv.171)

### 동작 규칙
- 필드에 배치됨. 4방향(북/남/동/서) 중 하나를 바라봄
- 카운터(숫자)를 가짐 (예: 5)
- Spawner가 바라보는 방향에 있는 보관함을 유저가 터치하면:
  - 카운터 -1 (5→4)
  - 큐에 새 보관함이 등장 (방출)
- 카운터 0이면 Spawner 비활성화 (더 이상 방출 없음)
- **투명/불투명은 파라미터:**
  - 투명 (Lv.41~): 다음에 방출될 보관함의 색상이 미리 보임
  - 불투명 (Lv.141~): 다음에 방출될 보관함의 색상을 알 수 없음
- **한 레벨 내 혼합 금지:** Spawner는 전부 투명 or 전부 불투명. 같은 레벨에서 투명+불투명 혼합 사용 불가
- 기믹 종류 수 카운트 시 **1종으로 취급**

### 난이도 레버
- Spawner 수
- 카운터 크기 (3 vs 5 vs 10)
- 방향 (필드 구조에 따라 유저 동선 영향)
- 방출될 보관함의 색상 구성
- 투명/불투명 선택 (불투명이 상위 난이도)

### 디자인 의도
- 큐 물량 압박. 유저가 보관함을 사용할 때마다 큐에 새 보관함이 추가되어, 홀더 초과 위험이 점진적으로 증가
- 투명: "다음에 뭐가 온다"를 보고 선행 계획 가능. 학습 부담 완화
- 불투명: "다음에 뭐가 올지 모른다". 불확실성 추가. 투명에서 학습한 규칙 그대로 적용
- Hidden + Spawner 불투명 조합 시 극한 불확실성

### 피드백
- 이펙트: 큐에 보관함이 팝 하고 등장
- 사운드: 톡! (등장음)
- **P2**

## 5. Pin (핀)

**도입**: Lv.61 (PKG 4) | **카테고리**: 필드 | **PF 대응**: Door (Lv.55)

### 동작 규칙
- 색상을 가진 비풍선 장애물. 1×N 형태로 필드를 가로막음
- 같은 색 다트로 직접 타격 시 한 칸씩 점진 제거 (앞에서부터 줄어듦)
- 전부 제거되면 가리고 있던 풍선이 노출됨
- 1:1 매칭 유지: 같은 색 다트 → 같은 색 핀 칸

### 난이도 레버
- 핀 길이 (1×3 vs 1×5 vs 1×8)
- 핀 색상 (필드에서 경쟁하는 다른 풍선과 같은 색이면 자원 경합)
- 핀 수
- 배치 위치 (핵심 풍선 앞을 가리는지)

### 디자인 의도
- 공간 제약 (필드, 점진 해제). 풍선이 아닌 비풍선이므로 다중 히트가 자연스러움
- 해당 색 다트를 핀 제거에 소비해야 하므로, 같은 색 풍선 처리와 자원 경합 발생
- Pin의 점진 제거 → 뒤의 풍선이 하나씩 드러나는 "점진 노출" 전략

### 피드백
- 이펙트: 핀 칸이 하나씩 제거되며 줄어드는 연출
- 사운드: 딸깍! (핀 제거음)
- **P2**

## 6. Lock & Key (자물쇠 & 열쇠)

**도입**: Lv.81 (PKG 5) | **카테고리**: 양쪽 | **PF 대응**: Locks and Keys (Lv.40)

### 동작 규칙
- 필드에 자물쇠 풍선 존재 (잠긴 상태. 다트가 맞아도 터지지 않음)
- 큐에 열쇠 보관함 존재 (열쇠 아이콘 표시)
- 열쇠 보관함을 레일에 보내면 자물쇠가 해제됨 (다트 발사 없이 잠금 해제만 수행)
- 해제 후 자물쇠 풍선은 일반 풍선과 동일하게 매칭 다트로 파괴
- 열쇠 보관함은 잠금 해제 후 제거됨 (레일에서 다트를 발사하지 않음)

### 난이도 레버
- 잠금 풍선 수
- 열쇠 보관함 위치 (큐의 뒷줄이면 먼저 꺼내기 어려움)
- 열쇠-자물쇠 색상 매칭 여부 (같은 색 열쇠만 해당 자물쇠 해제? or 범용 열쇠?)
- Hidden + Lock 조합 (열쇠가 숨겨져 있으면 찾기 어려움)

### 디자인 의도
- 순서 강제 (양쪽). 열쇠를 먼저 보내야 자물쇠 풍선을 터뜨릴 수 있으므로, 큐의 순서 관리가 강제됨
- "지금 열쇠를 보낼까, 다른 보관함을 먼저 처리할까"의 판단

### 피드백
- 이펙트: 자물쇠가 열리는 연출 + 풍선 색상 활성화
- 사운드: 찰칵! (자물쇠 열림)
- **P2**

## 7. Surprise Balloon (서프라이즈 풍선)

**도입**: Lv.101 (PKG 6) | **카테고리**: 필드 | **PF 대응**: Surprise Pixel (Lv.85)

### 동작 규칙
- 풍선의 색상이 ?로 숨겨져 있음
- 인접한 풍선이 터지면 색상이 공개됨
- 공개 후에는 일반 풍선과 동일 (같은 색 다트 1발로 팡)

### 난이도 레버
- Surprise Balloon 비율 (전체 풍선 중)
- 배치 위치 (중앙이면 공개 어려움. 외곽이면 먼저 공개 가능)
- Hidden(큐) + Surprise Balloon(필드) 조합 시 이중 불확실성

### 디자인 의도
- 정보 불확실성 (필드). Hidden(큐 정보 숨김)의 필드 버전
- "인접 풍선을 먼저 터뜨려야 색이 보인다" → 순서 제약 추가
- Hidden + Surprise 조합 시 큐도 필드도 안 보이는 극한 불확실성

### 피드백
- 이펙트: ? → 실제 색으로 전환
- 사운드: 띵! (공개음)
- **P2**

## 8. Wall (벽)

**도입**: Lv.121 (PKG 7) | **카테고리**: 필드 | **PF 대응**: Wall (Lv.120)

### 동작 규칙
- 필드에 고정된 파괴 불가 장벽
- 다트가 통과하지 못함
- 장벽 뒤의 풍선은 레일 경로 상 반대쪽에서만 타격 가능

### 난이도 레버
- 크기: 1×1 또는 2×2
- 위치 (핵심 풍선 앞이면 우회 필수)
- Wall 수 (여러 개 배치로 난이도 조절)

### 디자인 의도
- 공간 제약 (필드, 영구). Pin은 제거 가능하지만 Wall은 불가능
- "레일 한 바퀴 중 어느 구간에서 쏘느냐"의 공간 전략을 만듦
- Pin(점진 해제) vs Wall(영구 차단) 구분으로 공간 축이 풍부

### 피드백
- 이펙트: 다트가 장벽에 부딪혀 튕기는 연출
- 사운드: 딱! (단단한 충돌음)
- **P2**

## 10. Piñata Box (피냐타 상자)

**도입**: Lv.161 (PKG 9) | **카테고리**: 필드 | **PF 대응**: Egg Box (Lv.140)

### 동작 규칙
- Piñata(Lv.31)의 확장형. 2×2, 2×3 등 크기의 상자. 각 칸에 피냐타가 들어있음
- 각 피냐타는 색상 + 라이프(기본 10)를 가짐
- 같은 색 다트로 직접 타격 시 라이프 -1. 매 히트마다 금이 가는 연출
- 라이프 0 → 해당 칸의 피냐타 빠악 터짐
- 칸별로 색상이 다를 수 있고, 같을 수도 있음
- 모든 칸의 피냐타가 제거되어야 아래 필드가 노출됨
- 전부 제거될 때까지 아래를 가리는 역할

### 난이도 레버
- 크기 (2×2 vs 2×3 vs 3×3)
- 칸별 라이프 (5 vs 10 vs 15)
- 칸별 색상 조합 (같은 색 = 집중 소비, 다른 색 = 분산 소비)
- Piñata Box 수
- 아래 가려진 풍선의 중요도

### 디자인 의도
- 자원 압박 + 공간 제약. 대량의 다트를 소비하면서 동시에 필드를 가림
- 풍선(팡!) vs 피냐타(쾅쾅→빠악) 피드백 차별화
- Piñata(Lv.31)가 기본형이라면, Piñata Box는 "상자 안에 여러 칸 피냐타 + 가리는 역할"의 확장형. 단일 피냐타에서 학습한 "쾅쾅→빠악"을 대규모로 경험

### 피드백
- 이펙트 (타격): 금 가는 균열 + 흔들림
- 사운드 (타격): 쾅! (둔탁한 타격음)
- 이펙트 (파괴): 빠악 터지며 색종이 폭발 + 가려진 풍선 노출
- 사운드 (파괴): 빠악! + 축하 사운드
- **P1** (파괴 이펙트), **P2** (타격 이펙트)

## 11. Ice (얼음)

**도입**: Lv.201 (PKG 11) | **카테고리**: 필드 | **PF 대응**: Ice Blocks (Lv.200)

### 동작 규칙
- 필드의 특정 영역을 덮는 얼음 레이어. 색상 없음
- 라이프를 가짐 (예: 10)
- 다트로 직접 타격하는 것이 아님. **간접 제거**: 필드에서 어떤 풍선이든 제거될 때마다 라이프 -1
- 라이프 0 → 얼음이 깨지며 아래 필드가 노출됨
- 얼음 아래의 풍선은 얼음이 깨지기 전까지 보이지 않고 타격 불가

### 난이도 레버
- 라이프 (10 vs 20 vs 30)
- 얼음 크기 (덮는 영역)
- 얼음 수
- 아래 가려진 풍선의 중요도

### 디자인 의도
- 간접 제거 (필드). 1:1 매칭 규칙 완전 유지. 다트는 풍선에만 쓰이고, 얼음은 부수적으로 줄어듦
- "풍선을 많이 터뜨리면 얼음도 녹는다" — 직관적이고 풍선 세계관에 자연스러움
- Color Curtain(특정 색만 카운트)보다 단순하므로 먼저 도입

### 피드백
- 이펙트: 라이프 감소 시 얼음에 금이 감. 라이프 0에서 파삭 깨짐
- 사운드: 파삭! (얼음 깨짐)
- **P2**

## 12. Frozen Dart (얼어붙은 보관함)

**도입**: Lv.241 (PKG 13) | **카테고리**: 큐 | **PF 대응**: Frozen Shooter (Lv.290)

### 동작 규칙
- 보관함이 얼어있어 터치 불가
- 큐에서 인접한 보관함(좌우)을 사용하면 해동됨
- 해동 후 일반 보관함과 동일하게 동작

### 난이도 레버
- Frozen 보관함 수
- 위치 (큐의 가운데면 좌우 모두 써야 해동)
- Hidden + Frozen 조합 (숨겨진 상태 + 얼어붙은 상태)

### 디자인 의도
- 순서 강제 (큐). Hidden은 "안 보여서 못 고름", Frozen은 "보이지만 못 씀"
- 해동 순서가 강제되어 큐 관리 복잡도를 올림
- "옆을 먼저 보내야 이걸 쓸 수 있다"는 의존 관계

### 피드백
- 이펙트: 얼음 깨지는 연출
- 사운드: 파삭! (얼음 깨짐)
- **P2**

## 13. Color Curtain (컬러 커튼)

**도입**: Lv.281 (PKG 15) | **카테고리**: 필드 | **레퍼런스**: Yarn Loop: Curtain

### 동작 규칙
- 필드의 특정 영역을 덮는 커튼. **색상 + 카운터**를 가짐 (예: 빨강, 카운터 50)
- 다트로 직접 타격하는 것이 아님. **간접 제거**: 필드에서 **해당 색** 풍선이 제거될 때마다 카운터 -1
- 카운터 0 → 커튼 제거 → 아래 가려진 풍선 노출
- 커튼 아래의 풍선은 커튼이 제거되기 전까지 보이지 않고 타격 불가
- 1:1 매칭 규칙 완전 유지. 다트는 풍선에만 쓰이고, 커튼은 부수적으로 줄어듦

### 난이도 레버
- 카운터 크기 (20 vs 50 vs 100)
- 커튼 색상 (희소 색이면 카운터 줄이기 어려움)
- 커튼 수 (1개 vs 여러 개)
- 여러 커튼의 색상 (같은 색 vs 다른 색)
- 아래 가려진 풍선의 중요도

### 디자인 의도
- 간접 제거 (필드, 색상 종속). Ice(아무 풍선 제거 시 감소)의 상위 버전
- "빨강 커튼을 빨리 열려면 빨강 풍선을 우선 처리해야 한다" → 색상별 우선순위 판단 추가
- 마지막 기믹(Lv.281)으로 최종 복잡도. 필드 전체의 색상 제거 순서를 역으로 계획해야 함
- Ice(단순 간접) → Color Curtain(색상 종속 간접)의 자연스러운 학습 커브

### 피드백
- 이펙트: 카운터 감소 시 커튼이 점점 투명해짐. 카운터 0에서 커튼이 사라지는 연출
- 사운드: 스르륵~ (커튼 열림)
- **P2**

## 상호작용 매트릭스 (핵심 조합)

| 조합 | 효과 | 디자인 의도 |
|------|------|-----------|
| Hidden + Surprise Balloon | 큐도 필드도 안 보임 | 이중 불확실성 극한 |
| Hidden + Frozen Dart | 숨겨진 보관함 + 얼어붙은 보관함 | 큐 자유도 극한 제한 |
| Chain + Spawner | Chain 발동 시 Spawner가 추가 보관함 방출 | 홀더 초과 압박 급증 |
| Spawner(불투명) + Hidden | 기존 보관함도 추가 보관함도 색상 모름 | 큐 전체 불확실성 |
| Pin + Wall | 한쪽은 점진 제거 가능, 한쪽은 영구 차단 | 우회 경로 판단 필요 |

| Ice + Color Curtain | 아무 풍선 제거→Ice 감소, 특정 색 제거→커튼 감소 | 색상 우선순위 vs 총량 딜레마 |
| Piñata Box + Wall | Wall 뒤에 Piñata Box 배치 | 우회+다중타격 이중 부담 |

## AI 지시사항
- 기믹 도입 레벨과 PKG는 위 테이블 준수. 해당 PKG 이전 레벨에서 기믹이 등장하면 안 됨
- Chain의 연결 수(2/3/4)는 파라미터. 기믹 종류 수 카운트 시 1종으로 취급
- Spawner의 투명/불투명은 파라미터. 기믹 종류 수 카운트 시 1종으로 취급. **한 레벨 내 투명+불투명 혼합 금지**
- 레벨당 기믹 최대 5종 (Hard Rule)
- 기믹 6레벨 이상 연속 등장 금지 (Soft Rule)
- 기믹 사용 후 30레벨 이내 재등장 (Soft Rule, 망각 방지)
- 직접 타격 기믹(Pin, Piñata Box): 같은 색 다트 1:1 매칭 유지
- 간접 제거 기믹(Ice, Color Curtain): 다트는 풍선에만 소비. 풍선 제거의 부수 효과로 감소
- 피드백 우선순위: P0(Standard Balloon 터짐) > P1(Piñata/Piñata Box 파괴) > P2(기믹별 인지)
- MS(Magic Sort) 인게임 참조 금지. 아웃게임만 참조 가능. 인게임 레퍼런스는 PF + Yarn Loop만
